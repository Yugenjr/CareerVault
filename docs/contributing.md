# Contributing

Thanks for helping improve CareerVault.

## Before You Start

- Read the docs in this folder first.
- Check whether the change belongs in the frontend, backend, ML service, or docs.
- Keep changes focused and avoid unrelated refactors.

## Suggested Workflow

1. Create a branch for your work.
2. Make the smallest change that solves the problem.
3. Verify the affected service or UI path locally.
4. Update documentation if behavior changes.
5. Open a pull request with a clear summary and validation notes.

## Code Standards

- Preserve the existing architecture unless there is a clear reason to change it.
- Keep API responses and folder behavior consistent with the current app flow.
- Favor readable code over clever code.
- Add or update tests when the change affects behavior.

## Documentation Changes

- Update the docs folder when user-visible behavior changes.
- Include flow changes, new configuration values, or new setup steps.
- Add diagrams when a change affects the main upload or storage path.
